# 문제상황
1.  11/13:00 html 작성 시 `input`과`button`을 감싸는 그룹태그를 `form`태그가 아닌 `div`로 작성.
    * 사용자가 입력하는 값을 서버가 받아야 하기 때문에 form으로 태그 수정
2. 11/13:03 `form`을 사용한 후 `fieldset`과 `legend` 미입력
    * 추후 유지보수성을 위해 추가 후 클래스와 그룹제목 입력
3. 12/2:52 `text-ailgn`이 아닌 `display`를 `title`에 적용 
    * `text-ailgn`의 경우 텍스트와 인라인요소에만 적용가능
    * `display`의 경우 정렬하고자 하는 자식의 부모에게 부여하는 속성행
4. 12/13:15 `em {}`이 아닌 `em * {}`로 작성 후 글자 굵기 변화 없음 확인.
    * `em {}`은 `.popup_wrap` 안에 있는 모든 em을 선택하겠다는 의미
    * `em * {}`은 em의 모든 자식(popup_wrap의 자손까지)들을 선택하겠다는 의미가 됨.
5. 12/13:50 `document.querySelector()`미입력, 입력 후 변수 작성.
    * `document.querySelector()`는 HTML 파일에서 ()안에 값을 찾아 JS에 가져온다는 뜻.
    * `birthday_month.value = '12';`이 아닌 value를 기입하지 않고 값 12작성하여 출력되지 않는 오류가 발생. 위 내용을 해석하면 birthday_month가 가지는 속성 .value에 12라는 값을 넣는다라는 의미가 되므로 실제 사용자가 입력하는 값인.value 속성을 넣어 js를 수정함.
